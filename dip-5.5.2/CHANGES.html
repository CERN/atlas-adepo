ImprovementDIP-115Switch logging libraries from LOG4CXX to LOG4CPLUS<p>Apache Log4CXX was introduced in order to provide fine-grained logging support for the DIP API.</p> <p>However, this dependency introduced various problems :</p> <ul> <li>the Log4CXX source code has not been updated since 2009</li> <li>Log4CXX header files are platform-specific ( Windows / Linux ) - so build dependencies are all platform-specific too.</li> <li>The official LINUX build procedure for Log4CXX does not work on Linux SLC5 (as the standard LIBTOOL version is not compatible with the build, the build only works for SLC6, but we need an SLC5 version too)</li> <li>non-official Windows 64 bits builds are available on the Internet.</li> <li>Log4CXX on Linux imposes the deployment of further libraries (APR, APR Util, Xerces)</li> <li>Pre-built versions of Log4cxx for SLC5 use symbolic links and naming conventions that feature version numbers (for instance liblog4cxx.so.10), which are incompatible with JNLP (Java Webstart) - as JNLP only unpacks Linux libraries with a .SO extension and refuses to consider any other file as a native binary.</li> </ul> <p>By opposition log4CPLUS :</p> <ul> <li>... does not depend on anything but standard C++ libraries.</li> <li>... builds out of the box on SLC5, SLC6 32 / 64 bits and Windows 7 64 bits.</li> <li>... is largely compatible with Log4CXX (same configuration file format, same capabilities, better stability).</li> </ul> <p>We therefore need to replace LOG4CXX with LOG4CPLUS.</p>
